原型模式

Java24K 2022-09-14

# 一.原型模式介绍

概念：

  • 通过给出一个原型对象来指明所有创建的对象的类型，然后用复制这个原型对象的办法创建出更多同类型的对象。

原型模式特点：

  • 用于创建重复的对象，同时又能保证性能。

# 二.原型模式作用

  1. 基本就是你需要从A的实例得到一份与A内容相同，但是又互不干扰的实例的话，就需要使用原型模式。

# 三.原型模式类图与角色

原型模式角色

  1. 客户(Client)角色：客户类提出创建对象的请求。
  2. 抽象原型(Prototype)角色：这是一个抽象角色，通常由一个Java接口或Java抽象类实现。此角色给出所有的具体原型类所需的接口。
  3. 具体原型（Concrete Prototype）角色：被复制的对象。此角色需要实现抽象的原型角色所要求的接口。

# 四.原型模式实现

  • 具体原型 ： 实现了Cloneable接口, 并重写了Object(抽象原型)的clone()方法。
@Data
public class Prototype implements Cloneable{

    private String name;

    private int age;

    private String sex;

    public Prototype(String name, int age, String sex) {
        this.name = name;
        this.age = age;
        this. sex = sex;
    }


    /**
     * 重写object的clone()方法, 并将其作用域设置为public
     * @return
     * @throws CloneNotSupportedException
     */
    @Override
    public Object clone() throws CloneNotSupportedException {
        return super.clone();
    }

    @Override
    public String toString() {
        return "Prototype{" +
                "name='" + name + '\'' +
                ", age=" + age +
                ", sex='" + sex + '\'' +
                '}';
    }
}
  • 客户角色
public class PrototypeTest {

    public static void main(String[] args) throws CloneNotSupportedException {
        Prototype prototype = new Prototype("张三", 8, "男");
        Prototype cloneObject = (Prototype)prototype.clone();

        System.out.println(cloneObject);
    }
}
